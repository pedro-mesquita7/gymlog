# Plan 08-04: UI Primitive Components (Button, Input, Card)

**Requirements:** UI-02
**Depends on:** 08-02 (uses design tokens for consistent styling)

## Goal

Extract reusable Button, Input, and Card primitive components into `src/components/ui/`. Audit the codebase for duplicated patterns and create variant-driven primitives. Migrate the three specified components (StartWorkout, SetLogger, TemplateCard) to use the new primitives.

## Tasks

1. **Create `src/components/ui/Button.tsx`**
   - Variant props: `variant` ('primary' | 'secondary' | 'ghost' | 'danger'), `size` ('sm' | 'md' | 'lg')
   - `primary`: accent bg, black text (matches current "Start Workout" / "Log Set" buttons)
   - `secondary`: zinc-800 bg, zinc-100 text (matches current "Dismiss" style)
   - `ghost`: transparent bg, hover bg-zinc-800 (for menu items, icon buttons)
   - `danger`: red-400 text variant (for delete actions)
   - `size`: sm (py-2 px-3 text-sm), md (py-2.5 px-4), lg (py-4 text-lg font-bold, full-width)
   - Support `disabled` state with opacity-50 + cursor-not-allowed
   - Forward all native button props via `ComponentPropsWithoutRef<'button'>`
   - Use `className` merge (simple string concatenation is fine, no need for clsx/cn unless already installed)

2. **Create `src/components/ui/Input.tsx`**
   - Wraps `<input>` with consistent styling: bg-tertiary, border-secondary, rounded-lg, focus:ring-accent
   - Also export a `Select` variant wrapping `<select>` with same styling
   - Forward all native input/select props
   - Consistent with existing `bg-zinc-800 border border-zinc-700 rounded-lg` pattern but using design tokens

3. **Create `src/components/ui/Card.tsx`**
   - Simple container: bg-secondary/50, rounded-lg, padding
   - Variant: `variant` ('default' | 'interactive') â€” interactive adds hover state and cursor-pointer
   - Optional `className` prop for extending

4. **Migrate StartWorkout.tsx**
   - Replace inline `<select>` elements with `<Select>` primitive
   - Replace "Start Workout" `<button>` with `<Button variant="primary" size="lg">`
   - Keep all existing logic unchanged

5. **Migrate SetLogger.tsx**
   - Replace "Log Set" `<button>` with `<Button variant="primary" size="lg">`
   - Do NOT migrate NumberStepper's internal buttons (they have custom sizing logic)
   - Keep all existing logic unchanged

6. **Migrate TemplateCard.tsx**
   - Replace outer `<div>` wrapper with `<Card>`
   - Replace menu `<button>` items with `<Button variant="ghost">`
   - Replace delete button with `<Button variant="danger">`
   - Keep all existing logic unchanged
