# Plan 08-02: Design Token System & Geist Fonts

**Requirements:** UI-01
**Depends on:** None

## Goal

Establish a proper design token system in Tailwind CSS 4 using the `@theme` directive. Install and configure Geist Sans + Geist Mono fonts. Evolve the current zinc + orange palette into a modern, systematized dark theme inspired by Vercel Dashboard aesthetics. Remove the unused `App.css` boilerplate.

## Tasks

1. **Install Geist fonts**
   - `npm install @fontsource/geist-sans @fontsource/geist-mono`

2. **Create font imports** at `src/styles/fonts.css`
   - Import Geist Sans weights: 400, 500, 600, 700
   - Import Geist Mono weights: 400, 500, 600
   - These are self-hosted via @fontsource (no CDN)

3. **Redesign `src/index.css` with @theme directive and design tokens**
   - Import fonts.css
   - Use Tailwind CSS 4 `@theme` block to define:
     - **Colors**: Evolve zinc + orange. Define semantic tokens:
       - `--color-bg-primary`: zinc-950 (page bg)
       - `--color-bg-secondary`: zinc-900 (card bg)
       - `--color-bg-tertiary`: zinc-800 (input bg, hover states)
       - `--color-border-primary`: zinc-800
       - `--color-border-secondary`: zinc-700
       - `--color-text-primary`: zinc-100
       - `--color-text-secondary`: zinc-400
       - `--color-text-muted`: zinc-500
       - `--color-accent`: keep orange but use modern value (e.g., oklch for vibrancy)
       - `--color-accent-hover`: slightly dimmed accent
       - `--color-success`: green for PRs/positive indicators
       - `--color-error`: red for errors/destructive
       - `--color-warning`: amber for warnings
       - Chart colors: keep existing `--chart-*` variables, migrate to @theme
     - **Typography**: Font families via @theme
       - `--font-sans`: `'Geist Sans', system-ui, sans-serif`
       - `--font-mono`: `'Geist Mono', ui-monospace, monospace`
     - **Spacing**: Systematize if needed (Tailwind defaults are fine, just alias key values if helpful)
     - **Border radius**: `--radius-sm`, `--radius-md`, `--radius-lg` tokens
   - Update base layer to use Geist Sans as default font
   - Keep existing `.text-accent`, `.bg-accent`, `.border-accent` utility classes but update to use new token values

4. **Delete `src/App.css`** (Vite boilerplate, unused)
   - Remove the import from `src/App.tsx` if it exists (check first)
   - Note: App.tsx does NOT import App.css based on code review, so just delete the file

5. **Verify font loading** by checking that the app renders with Geist fonts in the browser (visual check during execution)
